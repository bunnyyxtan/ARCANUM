# Governance

User wallets are fork-only by design. GuardedWallet bytecode is immutable after deployment, so protocol evolution happens through new deployments, new modules chosen by owners, and open-source releases operators can inspect before adopting.

## What can change

- SDK ergonomics
- Dashboard views
- Indexer schema
- Optional modules
- New factory deployments

## What stays owner-controlled

- Wallet custody
- Signer authorization
- Policy updates
- Module rotation
- Unfreeze authority

This keeps Arcanum aligned with the permissionless nature of Arc.
