# Self-host Arcanum

Arcanum is a protocol, not a SaaS. Your data, your keys, your control.

Docker is not required for the current Arcanum web/API/Supabase workflow.
The primary path is npm for local development, Supabase through environment
variables, and Vercel-style deployment for the web app.

## Primary web workflow

```bash
npm install
npm run typecheck
npm run build
npm run lint
npm --workspace @arcanum/web run build
```

Configure Supabase, Arc RPC, SIWE, and telemetry values through environment
variables. Never commit `.env.local`, service role keys, or private keys.

## Environment

```txt
ARC_TESTNET_RPC=https://rpc.testnet.arc.network
ARC_TESTNET_WS=wss://rpc.testnet.arc.network/ws
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=replace-with-anon-placeholder
SUPABASE_SERVICE_ROLE_KEY=replace-with-server-only-placeholder
SIWE_SECRET=REPLACE_WITH_32_PLUS_CHAR_RANDOM_SECRET
ARCANUM_DEPLOYMENT_MODE=self-hosted
ARCANUM_TELEMETRY=off
```

Use managed Supabase backups and point-in-time recovery for production-like
testnet pilots. The repository does not maintain a local container database
backup command.

## Optional Helm chart

The Helm chart is optional infrastructure for operators that already run
Kubernetes. It is not required for local development or Vercel web deploys.

```bash
helm upgrade --install arcanum ./charts/arcanum \
  --set env.ARCANUM_TENANT_ID=default \
  --set ingress.host=arcanum.example.org
```

The chart supports ingress, cert-manager hooks, and external database/cache
configuration through values. Add `SIWE_SECRET` and other private values only in
your hosting provider's secret manager.
